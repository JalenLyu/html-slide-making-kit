Repeat Greeting Prevention | Detect only the first agent message and block repeated openings with override

1Current issue

Repeated greeting exposure

Auto Greeting has already greeted the user, but agents may still send a similar opening message in a short time.

Robotic first contact

Generic greetings such as "Hi, how can I help?" do not move the conversation toward issue handling.

Need conservative blocking

The detector should avoid blocking useful first replies that include greeting plus issue handling.

2New detection design

Composer hint

Agent sees a non-blocking reminder after Auto Greeting starts.

LLM3 check

Run detection only on the first human CS text before sending.

P2 block

If repeat greeting is detected, show reminder with Still Sending.

Guardrails: 2.5s timeout, fail-open on timeout/failure/invalid result, and region-level feature switch for rollout and rollback.
