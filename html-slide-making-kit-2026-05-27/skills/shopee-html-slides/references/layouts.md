# Shopee HTML Slide Layouts

Choose layouts based on the content's communication job, not visual variety.

## Core Layouts

| Layout | Use For | Requirements |
|---|---|---|
| `cover` | Deck title, date, team, confidentiality marker | Clear title and brand signal. |
| `executive-summary` | 3-5 key conclusions | Each point needs implication, not only observation. |
| `section-tracker` | Agenda and chapter progress | Active section highlighted in orange; inactive sections muted. |
| `two-column` | Problem/solution, context/objective, before/after | Balanced information and aligned headings. |
| `chart-with-insight` | Metric trend, comparison, impact evidence | Title states takeaway; chart supports the title. |
| `findings-list` | Multiple findings or discussion points | Numbered findings with concise evidence. |
| `framework-flow` | SCR, WWWH, Pyramid Principle, process flow | Direction and hierarchy must be obvious. |
| `venn-or-overlap` | Relationship between factors | Diagram clarifies the concept; labels remain readable. |
| `checklist` | QA rules, delivery checklist, acceptance criteria | Group checks logically; make scan path clear. |
| `before-after-review` | Positive/negative examples, rewrite practice | Separate source and improved version; annotations should not overload the slide. |

## Deck Structure Pattern

For business reviews and project summaries:

1. Cover
2. Executive Summary
3. Section Tracker
4. Context and Objectives
5. Background Facts and Data
6. Findings and Discussion Points
7. Conclusion and Next Steps
8. Appendix or Checklist

## Content Rules

- Start with the audience's decision or understanding need.
- Make slide titles assertions whenever possible.
- Avoid wall-of-text slides; convert paragraphs into evidence blocks, tables, charts, or grouped callouts.
- Keep each slide focused on one message.
- Use tracker slides when the deck has multiple sections or needs executive navigation.
